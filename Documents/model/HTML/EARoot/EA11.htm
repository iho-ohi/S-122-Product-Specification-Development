S-122 uses the <b>associatedRXN </b>association to link <b>NauticalInformation </b>with feature types, like other sub-types of <b>AbstractRXN</b>. S-101 defines only <b>NauticalInformation </b>and uses its standard <b>additionalInformation </b>association.<br/><br/>The problem with using <b>additionalinformation </b>everywhere is that you have to access the target object before you know its type - whether it is <b>Regulations</b>, <b>Restrictions</b>, <b>Recommendations</b>, <b>NauticalInformation</b>, or some other info type like <b>ContactDetails</b>, <b>Authority</b>, etc.<br/>
